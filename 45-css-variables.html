Variables en CSS

Se definen con dos guiones (--)nombre-de-la-variable

Se usan:

    tipo-selector {
      propiedad: var(--nombre-de-la-variable)
    }

Variables globales

Pueden ser usadas en cualquier lugar de la página

Se definen dentro del selector :root

    ---CSS
    :root{
      --margen: 10px;
    }

    ul{ 
      margin-left: var(margen); /* ← Variable global */
      }
      

    ---HTML
   <ul>
    <li>...</li>
    <li>---</li>
    <li>***</li>
   </ul>

Variables en contexto definido

Se aplicarán en los elementos hijos del elemento en el cual ha sido definido,

En otras palabras, si se define una variable dentro de un selector de clase, el elemento HTML debe poseer esa clase para que se efectue la sustitución del valor de dicha variable


    ---CSS
    
    /* Variables en contexto definido */
    .tool-box{
      --rojo: rgb(255, 0, 0);
      --verde: rgb(00, 255, 0);
      --azul: rgb(00, 00, 255);
      --chocolate: rgb(122, 62, 5);
      --grande: 50px;
      --mediano: 30px;
      --pequeno: 10px;
    }

    /* Selectores */

    #rojo-grande{
      color: var(--rojo, purple);
      font-size: var(--grande);
    }
    
    .verde-mediano{
      color: var(--verde,purple );
      font-size: var(--mediano);
    }
    
    [attr="azul-pequeno"]{
      color: var(--azul, purple);
      font-size: var(--pequeno);
    }
    
    [attr="text-chocolate"]{
      color: var(--chocolate);
    }
    
    
    ---HTML
    <ul>
      <li class="tool-box" id="rojo-grande">rojo-grande</li>
      <li class="tool-box verde-mediano">verde-mediano</li>
      <li class="tool-box" attr="azul-pequeno">azul-pequeno</li>
      <li class="tool-box" attr="text-chocolate">chocolate</li>
    </ul>

    <ul class="tool-box">
      <li  id="rojo-grande">rojo-grande</li>
      <li class="verde-mediano">verde-mediano</li>
      <li  attr="text-chocolate">chocolate</li>
      <li  attr="azul-pequeno">azul-pequeno</li>
    </ul>

Aplicando la clase tool-box a cada elemento <li>

  • rojo-grande
  • verde-mediano
  • chocolate
  • azul-pequeno

Aplicando la herencia de clase tool-box en <ul>

  • rojo-grande
  • verde-mediano
  • chocolate
  • azul-pequeno

Compatibilidad entre navegdores

Si un navegador como p.e. Google Chrome es capaz de interpretar variables CSS, entonces, supponiendo que establezcamos una variable para el color podría ser utilizada así:

    ---CSS
    .paleta-color{
      --rojo: rgb(255, 0, 0);
    }

    #texto-rojo{
      color: var(--rojo, color-de-respaldo);
    }

En el ejemplo anterior si por algún motivo (escribir mal el nombre de la variable: --roja) se utilizaria el color de respaldo para mostrar lo que queremos hacer.

Cuando un navegador no interpreta variables CSS la solución será incluir la misma propiedad antes de la propiedad que utilizó de la variable

    ---CSS
    .paleta-color{
      --rojo: rgb(255, 0, 0);
      --verde: rgb(0, 255, 0);
    }

    /*CASO A: sin color de respaldo*/
    #texto-rojo{
      color: var(--rojo, color-de-respaldo);
      color: var(--roja);
    }

    /*CASO B: sin color de respaldo*/

    .texto-verde {
      color:rgb(0, 255, 0)
      color:var(--verde, green)
    }

Es necesario seguir utilizando el color-de-respaldo ya que si se omite y has escrito mal el nombre de la variable no se mostrará color alguno (CASO A) ya que:

  1. El navegador leerá e interpretará la propiedad:

            color:rgb(255, 0, 0);

  2. Se interpretará la propiedad:

            color:var(--roja)

    Pero ya que la función var() en la propiedad ha sido intepretada pero el nombre de la variable ha sido mal escrito y no existe un color de respaldo y, además, CSS aplicará la última propiedad válida, entonces, no se mostrará adecuadamente en el navegador.
