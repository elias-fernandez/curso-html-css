Prioridades en CSS

Prioridades en presencia de CSS inline, CSS interno y CSS externo

    ---CSS interno
    <head>
      <style>
        p {
          color: green;
        }
      </style>

      <link rel="stylesheet" href="assets/css/42-prioridad-css-2.css" type="text/css">
    </head>


    ---CSS externo
    p {
      color: mediumslateblue;
    }
    
    #el-parrafo{
      color: crimson;
    }


    ---HTML
    <p style="color: blue;">a)Aplicando CSS inline</p>
    <p id="el-parrafo" style="color: blue;">b) Aplicando CSS inline + CSS externo</p>
    <p>c) Aplicando CSS interno</p>
    <p>d) Aplicando CSS externo</p>

a) Aplicando CSS inline

b) Aplicando CSS inline + CSS externo

c) Aplicando CSS interno

d) Aplicando CSS externo

En los casos a) y b) se aplica el estilo inline por tener más prioridad que el CSS interno, el externo o el ID

Miralo de esta forma:

Aunque poseea un ID y este posea un selector de archivo CSS, YO QUIERO QUE ESTE ID sea del color X, y eso es más específico ya que aplica el estilo al elemento que posee ese ID

En los casos c) y d) se aplica el estilo externo porque el CSS extern fue llamado en el <head> posteriormente a la declaración de css interno (<style>...</style>) y al tratarese de elemento <p> en ambas hojas de estilo aplica el case de selectores del mismo tipo con proìedades en conflicto
